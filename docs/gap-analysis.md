| 级别 | 含义 | 建议阶段 |
|------|------|---------|
| **P0** | 缺失即不可用的基础功能 | 立即 |
| **P1** | 显著影响用户体验的差距 | alpha 早期 |
| **P2** | 增值功能，可差异化竞争 | alpha 中期 |
| **P3** | 生态/客户端，投入大回报长 | 暂缓 |

## 9. Stora 差异化优势（竞品不具备）

这些是自部署带来的独特竞争力，应持续强化：

| 优势 | 说明 |
|------|------|
| 🏠 **自部署/数据主权** | 数据完全由用户掌控，无隐私泄露风险 |
| 🔓 **开源 (Apache-2.0)** | 可审计、可定制、零 vendor lock-in |
| 🌐 **WebDAV 协议** | 桌面/移动客户端通过 WebDAV 直接挂载 |
| 🔐 **端到端加密 Vault** | AES-256-GCM + PBKDF2，服务端不可解密 |
| 🤖 **MCP/AI 代理** | 内置 AI 代理操作文件系统，互联工具生态 |
| 📋 **完整审计日志** | 满足 SOC2/ISO27001 类合规要求 |
| 🧩 **S3 兼容存储** | 对接任意 S3 兼容存储（MinIO/S3/R2/Backblaze） |
| 🛠 **WOPI 协议** | 可对接 Microsoft Office Online / Collabora Online |
| 📦 **轻量部署** | 单机可运行，无微服务强制依赖 |
| 🆓 **免费无限用户** | 免费使用，无用户数限制 |

## 10. 下一步建议

1. **alpha 阶段优先实现 P1 差距项**（G1/G2/G3），快速补上基础体验短板
2. **alpha 中期实现 P2 差距项**中的分享增强（G4/G5/G6）和图片编辑（G8）
3. **团队空间（G7）** 是重量级需求，建议提前架构设计，分步落地
4. **Office Online 集成（G14）** 利用已有 WOPI 接口，部署配套服务即可
5. **持续强化差异化优势**：MCP 代理、Vault 加密、WebDAV 是自部署网盘的核心竞争力
