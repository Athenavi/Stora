# 分享功能分析报告

## 一、现有能力评估

### 功能覆盖矩阵

| 功能 | 当前状态 | 备注 |
|------|---------|------|
| **分享文件** | ✅ 基本完成 | 单文件分享链接正常工作 |
| **分享文件夹** | ❌ 不支持 | 只有 `file_id`，没有 `folder_id` 处理逻辑 |
| **密码保护** | ✅ 支持 | 后端支持，但前端批量分享未开放设置 |
| **过期时间** | ⚠️ 部分支持 | 后端支持 `expires_in_hours`，前端批量分享未开放 |
| **下载次数限制** | ⚠️ 部分支持 | `max_downloads` 已修复，前端未暴露 |
| **权限级别** | ⚠️ 基础 | read / download / edit 三级，但 edit 无实际实现 |
| **分享管理页** | ⚠️ 基础 | 列表 + 撤销 + 批量撤销，缺少编辑/筛选/搜索 |
| **分享接收页** | ⚠️ 基础 | 文件预览+下载，不支持文件夹浏览 |
| **用户间分享** | ⚠️ 存在但未用 | `file_shares` 表和 `ShareWithUser` 端点存在但前端无入口 |
| **访问统计** | ✅ 支持 | view_count / download_count 已实现 |
| **分享编辑** | ❌ 不支持 | 创建后无法修改密码/过期/权限 |
| **直接分享到用户** | ❌ 不支持 | 无法按用户名/邮箱分享 |

### 核心缺失：文件夹分享

`share_links` 表结构当前只有 `file_id`，要支持文件夹需要：
- 新增 `folder_id` 列（可为空，与 file_id 互斥）
- 访问时递归获取文件夹下所有文件
- 下载时打包 ZIP（可复用 `BatchDownload` 逻辑）
- 接收页面展示文件夹树形结构

## 二、业界标准功能矩阵

### Google Drive（参考基准）

| 功能 | 说明 |
|------|------|
| **分享范围** | 受限（指定用户）/ 知道链接的人 / 组织内 / 公开 |
| **权限** | 查看者 / 评论者 / 编辑者 |
| **文件夹分享** | 继承权限到子文件，支持嵌套 |
| **密码保护** | 仅企业版 |
| **过期时间** | 支持 |
| **下载禁止** | 查看者可禁止下载 |
| **分享管理** | 逐个查看、修改权限、移除 |
| **链接缩短** | 自动生成短链接 |
| **通知** | 被分享时邮件通知 |
| **分享到用户** | 按邮箱直接分享，协作编辑 |

### Dropbox / 百度网盘 / 阿里云盘 共性

| 功能 | 说明 |
|------|------|
| **提取码** | 百度网盘专利：4位数字提取码（非密码） |
| **有效期** | 1天 / 7天 / 永久（常用三种） |
| **文件夹分享** | 分享整个目录，接收方可下载或在线预览 |
| **分享链接编辑** | 创建后可修改有效期和密码 |
| **转存** | 接收方可一键保存到自己的网盘 |
| **水印** | 预览时自动添加查看者水印 |
| **上传分享** | 他人可上传文件到指定分享链接（收集模式） |

## 三、实践建议

### Phase 1 — 高价值低成本的修复（1-2天）

弥补当前最明显的缺失：

```
1. 文件夹分享（后端）
   └─ share_links 新增 folder_id 列（与 file_id 互斥，CHECK 约束）
   └─ CreateShareLink 接受 folder_id 参数
   └─ VerifySharePassword 查询 folder_id，有则递归获取文件列表
   └─ ShareFileDownload → 文件夹走 ZIP 打包（复用 BatchDownload 的 ZIP 流）

2. 分享管理页增强
   └─ 显示分享的文件/文件夹名称和类型图标
   └─ 显示分享链接的完整状态（已过期/即将过期/有效）
   └─ 筛选：仅显示有效/已失效
   └─ 搜索：按文件名搜索分享

3. 创建分享弹窗增强
   └─ 密码输入（当前批量分享直接跳过了）
   └─ 有效期选择：1天 / 7天 / 30天 / 永久
   └─ 权限选择：仅查看 / 可下载
```

设计决策：`folder_id` vs 独立 `share_folders` 表

| 方案 | 优点 | 缺点 |
|------|------|------|
| share_links.folder_id 可为空 | 改动最小，现有查询微调 | 共享表在列表查询时需 JOIN folders |
| 独立 share_folders 表 | 关注点分离 | 多一套 CRUD，前端也要维护两套 |
| **推荐：folder_id 列** | 数据模型简单，工作量少 | 无本质缺陷 |

### Phase 2 — 中等价值提升体验（3-5天）

```
4. 分享接收页增强 — 文件夹浏览
   └─ 文件夹：展示目录树，可点击进入子目录
   └─ 文件：预览缩略图或图标
   └─ 支持勾选多个文件批量下载 ZIP
   └─ 支持一键转存到自己的 Stora（需鉴权用户）

5. 分享链接编辑
   └─ 已创建的分享可修改：密码 / 过期时间 / 权限 / 是否激活
   └─ 对应后端 PUT /files/shares/{id} 端点

6. 分享到用户
   └─ 按用户名/邮箱搜索用户
   └─ 直接分配给指定用户（使用已有 file_shares 表）
   └─ 被分享者在"与我共享"页面查看
```

### Phase 3 — 高阶功能（按需）

```
7. 提取码（4位数码而非长密码）
8. 查看者水印
9. 上传收集模式（他人可上传文件到分享目录）
10. 分享数据看板（访问趋势、下载来源、设备分布）
```
