# Stora — 项目现状与未来规划

> 生成时间: 2026-06-17 | FastBlog V0.5.26.0617 → Stora 网盘重构

---

## 一、项目现状

### 1.1 代码库规模

| 维度 | 数据 |
|------|------|
| Python 文件 | ~340 个 |
| ORM 模型 | 33 个（8 模块） |
| API v2 模块 | 14 个（home, dashboard, search, notifications, media, files, security, auth, users, performance, system, compliance, admin/cache, mcp） |
| 前端页面 | 8 个（login, register, drive, share, trash, favorites, view, admin） |
| 数据库表 | 33 张（stora2606） |
| 存储驱动 | local 本地 + S3/MinIO/R2 |

### 1.2 已实现的功能

```
用户系统:   注册 / 登录(JWT) / 资料 / 2FA
文件管理:   CRUD / 上传(单文件+分片+秒传) / 下载(令牌) / 搜索/ 预览
文件夹:    CRUD / 树形 / 移动 / 导航面包屑
分享:      创建(密码+过期+下载限制) / 列表 / 撤销 / 访问
回收站:    列表 / 恢复 / 批量恢复 / 永久删除 / 清空
存储配额:  跟踪 / API 接口
系统:      设置 / 审计日志 / 通知
权限:      RBAC 角色 / Capability / 字段级权限
安全:      登录尝试 / Token 黑名单 / 敏感词 / GDPR
前端:      Qwik + Tailwind v4 / 设计系统 / 暗色模式 / 响应式
```

### 1.3 当前架构

```
┌─ Frontend (Qwik) ─────────────────────────────┐
│  Login | Register | Drive | Share | Trash      │
│  Favorites | View(Preview) | Admin             │
│  UI: Icon/Button/Input/Skeleton/Badge          │
└──────────────────────┬─────────────────────────┘
                       │ API calls (fetch + JWT)
┌─ Backend (FastAPI) ──┼─────────────────────────┐
│  src/api/v2/         │  14 route modules        │
│  src/api/v3/         │  mobile auth/media/users │
│  src/auth/           │  JWT + password hashing  │
│  shared/models/      │  33 ORM models           │
│  shared/services/    │  files, users, security  │
│  shared/services/files/drivers/  │ local + S3   │
└──────────────────────┬─────────────────────────┘
                       │ SQLAlchemy
┌─ Database ───────────┼─────────────────────────┐
│  PostgreSQL: stora2606  │  33 tables            │
└─────────────────────────────────────────────────┘
```

## 四、已知技术债务

| 问题 | 影响 | 建议修复时机 |
|------|------|-------------|
| Qwik lint 对闭包 handler 的误报 | 构建需跳过 lint | 优化 eslint 配置或重构为 $() 模式 |
| `unified_users.py` 含博客遗留代码 | 部分端点不适用网盘 | 逐步清理或重写 |
| `batch_operations.py` 引用已删 Article 模型 | 编译通过但运行时出错 | 重写为文件批量操作 |
| 缺少 `pyproject.toml` 依赖锁定 | 环境不一致 | 添加 pip freeze 或 poetry 锁定 |
| 前端未接入真实存储配额数据 | 侧边栏显示假数据 | P0.6 修复 |
| MCP tools 引用已删 article 模型 | 运行时出错 | 清理或删除 |
