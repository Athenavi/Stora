# Stora — 重构规划蓝图

> **战略目标**: 将 FastBlog（博客/CMS）彻底改造为 Stora（高性能网盘应用）
> **技术栈**: 后端 Python 3.14+ · FastAPI · SQLAlchemy 2.0 | 前端 Qwik · Tailwind CSS v4
> **生成时间**: 2026-06-17 | 当前状态: 规划阶段

---

## 一、模型保留/淘汰分析

### 1.1 决定矩阵

对 `config/models.yaml` 中111个模型（103 ORM + 8 Schema）逐一分析：

| 决策 | 数量 | 说明 |
|------|------|------|
| ✅ **保留** | ~20 | 用户体系、文件存储、基础系统、权限核心 |
| 🗑️ **删除** | ~83 | 博客内容、电商、VIP、协作、SEO、多站点等 |
| 🆕 **新增** | ~12 | 网盘特有模型（分享链接、回收站、存储配额等） |

### 1.2 ✅ 保留的模型（核心基石）

```
用户与安全:
  User, UserBlock, UserSession, LoginAttempt, TokenBlacklist
  Role, Capability, RoleCapability, UserRole, PermissionAuditLog
  FieldPermission, GDPRConsent
  
文件存储（核心）:
  Media → 重命名为 FileItem
  MediaFolder → 重命名为 Folder  
  FileHash → 重命名为 FileFingerprint
  MediaOptimization → 保留
  UploadTask, UploadChunk, DownloadTask → 保留

系统:
  AuditLog, SystemSettings, AdminSettings
  Notification
  SearchHistory
```

## 五、架构变化总览

### 当前 FastBlog 架构

```
FastBlog (Astro + React + FastAPI)
├── 前端: 82 页面、215 组件、28 API 服务
├── API: 1,273 端点、168 路由文件
├── 模型: 111 个、380 服务文件
└── 插件/主题: 11 插件、3 主题
```

### 目标 Stora 架构

```
Stora (Qwik + FastAPI)
├── 前端: Qwik City（~30 页面、~60 组件）
├── API: ~200 端点、~40 路由文件
├── 模型: ~45 个、~80 服务文件
└── 插件: 可选（文件预览引擎、存储后端）
```

### 核心变化

| 维度 | FastBlog | Stora |
|------|----------|-------|
| 前端框架 | Astro + React 19 | Qwik |
| 页面类型 | 博客前台 + 管理后台 | 网盘主界面 + 管理后台 |
| API 规模 | 1,273 端点 | ~200 端点 |
| 模型数 | 111 | ~45 |
| 主数据库表 | ~80+ | ~30 |
| 核心功能 | 文章发布/编辑/管理 | 文件上传/下载/分享/管理 |
| 存储 | 本地 + S3（媒体库） | 本地 + S3/MinIO（主存储） |
| 文件处理 | 缩略图、图片编辑 | 缩略图、预览、转码、加密 |
| 用户付费 | VIP 等级 | 存储空间套餐 |
| 第三方集成 | 博客迁移、SEO | 云存储挂载、WebDAV |

## 六、风险和注意事项

### 6.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Qwik 生态成熟度 | 组件库/工具链不如 React | 先跑 POC 验证关键功能 |
| 数据迁移 | 用户/媒体数据丢失 | 编写独立迁移脚本，先测试 |
| 删除力度过猛 | 可能误删需要的功能 | 每步删除后验证编译通过 |
| 并行开发冲突 | Phase 2/3/4 依赖关系 | 严格按 Phase 顺序执行 |

### 6.2 保留策略

以下功能应在删除前确认是否值得保留：

| 功能 | 决策 | 理由 |
|------|------|------|
| SSO/OAuth 登录 | ✅ 保留 | 网盘也需要多种登录方式 |
| 2FA 双因素认证 | ✅ 保留 | 文件安全需要 |
| RBAC 角色权限 | ✅ 保留 | 文件分享权限需要 |
| 审计日志 | ✅ 保留 | 安全合规需要 |
| 速率限制 | ✅ 保留 | 防止滥用 |
| Webhook | ⚠️ 暂保留 | 可能用于文件事件通知 |
| 通知系统 | ✅ 保留 | 文件分享/操作通知 |
| 全文搜索 | ✅ 保留 | 文件搜索核心功能 |
| 缓存系统 | ✅ 保留 | 性能关键 |
| 备份/恢复 | ✅ 保留 | 运维必需 |
| 安装向导 | ✅ 保留 | 首次部署引导 |
| API Key | ✅ 保留 | SDK/客户端认证 |
