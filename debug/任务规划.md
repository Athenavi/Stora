### 🟢 P2 — 质量提升（不含 API 文档、i18n）

#### P2.1: 错误处理统一

```
目标: 全局异常处理器、错误码规范、结构化日志

子步骤:
  [ ] P2.1.1  创建 src/utils/errors.py — 错误码枚举 + StoraException
               - 错误码: FILE_NOT_FOUND, PERMISSION_DENIED, QUOTA_EXCEEDED, UPLOAD_FAILED 等
               - StoraException(code, message, status_code)
  [ ] P2.1.2  创建 src/middleware/error_handler.py — 全局异常捕获中间件
               - 捕获 StoraException → JSON 响应
               - 捕获 ValidationError → 422 响应
               - 捕获 HTTPException → 透传
               - 捕获未预期异常 → 500 + 日志
  [ ] P2.1.3  替换所有 fail() 调用为 raise StoraException
  [ ] P2.1.4  统一日志格式: request_id, user_id, ip, method, path, duration
  [ ] P2.1.5  在 app.py 注册全局异常处理器
```

#### P2.2: 输入校验加固

```
目标: 文件名/路径校验、XSS 防护、文件类型白名单

子步骤:
  [ ] P2.2.1  文件名校验: 禁止特殊字符 (../, \0, 控制字符)
               - MAX_FILENAME_LENGTH = 255
               - 禁止空文件名、纯空格、Windows 保留名
  [ ] P2.2.2  路径穿越防护: upload.py 中校验 extract 后的路径不超出目标目录
  [ ] P2.2.3  文件类型白名单: 上传时校验签名 (magic bytes) 而非仅 MIME
               - 图片: JPEG/PNG/GIF/WebP
               - 文档: PDF/DOCX/XLSX
               - 脚本: 禁止 .exe/.bat/.sh/.js 等可执行文件（可配置开关）
  [ ] P2.2.4  XSS 防护: 文件名/描述中过滤 HTML 标签
  [ ] P2.2.5  速率限制: 上传/下载/登录端点增加速率限制
```

### 🟡 P3 — 功能增强

```

  [ ] P3.1.1  创建 shared/services/security/crypto.py
               - encrypt_file(key, data) → nonce + ciphertext + tag

  [ ] P3.1.2  密钥管理: 主密钥 (MASTER_KEY env) + 每文件衍生密钥
```
